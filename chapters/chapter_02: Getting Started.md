### 2.2.3 How to Install Ubuntu (Dual Boot, USB Live, Virtual Machine, Embedded Boards, and More)

Ubuntu is incredibly flexible ‚Äî you can install it almost anywhere. Whether you're using a Windows laptop, a MacBook, a Raspberry Pi, or even a cloud-based terminal, there‚Äôs a Linux environment waiting for you.

#### Option 1: USB Live Boot

This is a suggested option for beginners who want to explore Ubuntu safely. USB Live Boot lets you run Ubuntu directly from a USB stick without installing it or changing your hard drive. It's perfect for testing, learning, and experimenting with Linux tools like VS Code and nano in a temporary environment.

##### USB Live Boot: Step-by-Step Guide for Beginners

Want to try Ubuntu without installing it? Boot it from a USB stick! This method is safe, fast, and perfect for beginners who want to explore Ubuntu without changing their computer.

**Yes, you can install applications like VS Code, nano, and other CLI packages ‚Äî but only if you choose the "Try Ubuntu" option and have internet access.**

##### What You Need

* A USB stick (at least 8 GB)

* Ubuntu ISO file (download from [ubuntu.com](https://ubuntu.com/))

* A tool to create the bootable USB (Rufus for Windows, BalenaEtcher for macOS/Linux)

##### Step-by-Step Instructions

* **Download Ubuntu ISO**

  * Go to [ubuntu.com/download](https://ubuntu.com/download)

  * Choose the latest LTS version for stability

* **Create a Bootable USB**

  * Open Rufus or BalenaEtcher

  * Select the Ubuntu ISO and your USB drive

  * Click "Start" to write the ISO to the USB

* **Restart Your Computer and Enter Boot Menu**

  * Plug in the USB

  * Restart your computer

  * Press the boot key (usually F12, Esc, or Del)

  * Select your USB drive to boot

* **Choose "Try Ubuntu"**

  * When Ubuntu loads, select "Try Ubuntu" instead of installing it

  * This runs Ubuntu directly from the USB ‚Äî no changes to your hard drive

* **Explore Ubuntu Safely**

  * You‚Äôll see the Ubuntu desktop and can use the terminal, browser, and file manager

  * You can install apps like:

    * `nano` ‚Üí `sudo apt install nano`

    * `g++` ‚Üí `sudo apt install g++`

    * `VS Code` ‚Üí Download from [code.visualstudio.com](https://code.visualstudio.com/) or use `snap install code --classic`

* **Persistence (Optional)**

  * If you want Ubuntu to remember your changes (installed apps, saved files), create a **persistent USB**

  * Rufus and other tools offer this option during setup

##### What You Can Learn

* How Ubuntu works without installing it

* How to use the terminal and install packages

* How to write and run C++ code

* How to explore VS Code and other tools

##### When You‚Äôre Done

* Shut down Ubuntu

* Remove the USB

* Your computer will boot back into Windows/macOS like nothing happened

This is the safest way to explore Ubuntu. You can install apps, write code, and learn Linux ‚Äî all without touching your hard drive.


This setup lets you choose between Ubuntu and your existing OS every time you start your computer. It‚Äôs perfect for geek kids who want a dedicated Ubuntu system for coding, while keeping Windows or macOS for school or family use.

#### Option 2: Dual Boot with Windows or macOS

This setup lets you choose between Ubuntu and your existing OS every time you start your computer. It‚Äôs perfect for geek kids who want a dedicated Ubuntu system for coding, while keeping Windows or macOS for school or family use.

##### Dual Boot Ubuntu: Step-by-Step Guide for Beginners

Dual booting means installing Ubuntu alongside your current operating system (like Windows or macOS), so you can choose which one to use each time you start your computer. It‚Äôs powerful ‚Äî but risky if done carelessly. Follow these steps exactly, and **always back up your data first.**

##### Step 1: Understand What Dual Booting Does

* Your hard drive will be split into two sections (called partitions).

* One partition keeps your current OS (Windows/macOS), the other holds Ubuntu.

* At startup, you‚Äôll choose which OS to boot using a menu called GRUB.

‚ö†Ô∏è **WARNING:** If you accidentally delete or overwrite your current OS partition, you could lose all your files. That‚Äôs why the next step is critical.

##### Step 2: Back Up Everything

* Save important files to an external hard drive or cloud storage (Google Drive, Dropbox, etc.).

* Don‚Äôt skip this ‚Äî even pros make mistakes.

##### Step 3: Download Ubuntu

* Go to [ubuntu.com](https://ubuntu.com/download) and download the latest Ubuntu Desktop ISO file.

* Choose the LTS (Long Term Support) version for stability.

##### Step 4: Create a Bootable USB

* Use **Rufus** (Windows) or **BalenaEtcher** (macOS/Linux) to write the ISO to a USB stick.

* Plug in your USB and launch the tool.

* Select the Ubuntu ISO and your USB drive, then click ‚ÄúStart.‚Äù

##### Step 5: Shrink Your Existing Partition

* On Windows: Open Disk Management ‚Üí Right-click your main drive ‚Üí ‚ÄúShrink Volume.‚Äù

* On macOS: Use Disk Utility ‚Üí Select your drive ‚Üí Partition ‚Üí Add a new partition.

* Leave at least 20 GB free for Ubuntu.

‚ö†Ô∏è **WARNING: Do not delete any partitions. Only shrink.**

##### Step 6: Boot from USB

* Restart your computer and enter the boot menu:

    * Windows: Usually F12, Esc, or Del

    * macOS: Hold Option key

* Select your USB drive to boot into Ubuntu.

##### Step 7: Start Ubuntu Installation

* Click ‚ÄúInstall Ubuntu.‚Äù

* Choose **‚ÄúInstall Ubuntu alongside Windows/macOS‚Äù** ‚Äî this is the safest option.

* If you don‚Äôt see this option, STOP and ask for help. Manual partitioning is risky.

##### Step 8: Follow the Installer Carefully

* Select your language, keyboard layout, and time zone.

* When asked about partitions, **double-check** that your existing OS is not being erased.

* Confirm that Ubuntu is being installed in the new space you created.

##### Step 9: Finish Installation and Reboot

* Let Ubuntu install (takes 10‚Äì30 minutes).

* When done, **remove the USB** and restart your computer.

##### Step 10: Choose Your OS at Startup

* You‚Äôll see a menu called GRUB.

* Use arrow keys to select Ubuntu or Windows/macOS.

* Press Enter to boot.

##### You‚Äôre Done!

You now have a dual-boot system. Ubuntu is ready for coding, and your original OS is still intact.




#### Option 3: Virtual Machine (VM)

This is a beginner-friendly way to run Ubuntu inside your current operating system without changing anything on your hard drive. You‚Äôll use a virtual machine app like VirtualBox or VMware to create a "computer inside your computer" ‚Äî perfect for safe experimentation.

##### What You Need

* A computer running Windows or macOS

* At least 8 GB RAM and 20 GB free disk space

* Ubuntu ISO file (download from [ubuntu.com](https://ubuntu.com/download))

* VirtualBox (free) or VMware Workstation Player (free for personal use)

##### Step-by-Step Guide

* Download and Install VirtualBox

    * Go to [virtualbox.org](https://www.virtualbox.org/) and download the installer for your OS

    * Install it like any other app

* Download Ubuntu ISO

    * Visit [ubuntu.com/download](https://ubuntu.com/download)

    * Choose the latest LTS version for stability

* Create a New Virtual Machine

    * Open VirtualBox ‚Üí Click "New"

    * Name it "Ubuntu VM" and select "Linux" as the type, "Ubuntu (64-bit)" as the version

    * Allocate memory (at least 4096 MB recommended)

    * Create a virtual hard disk (20 GB or more)

* Attach the Ubuntu ISO

    * Go to Settings ‚Üí Storage ‚Üí Click the empty disk icon ‚Üí Choose the Ubuntu ISO file

* Start the VM and Install Ubuntu

    * Click "Start" to boot the VM

    * Follow the Ubuntu installer steps (language, keyboard, time zone, etc.)

    * Choose "Erase disk and install Ubuntu" ‚Äî this only affects the virtual disk, not your real computer

* Finish Installation and Reboot

    * Let Ubuntu install (takes 10‚Äì30 minutes)

    * Reboot the VM when prompted

* Explore Ubuntu Safely

    * You now have a full Ubuntu desktop inside a window

    * You can install apps like nano, g++, and VS Code

    * Try writing and compiling C++ code, using the terminal, and exploring Linux commands

##### What You Can Learn

* How Ubuntu works without installing it on your real computer

* How to use the terminal and install packages

* How to write and run C++ code

* How to experiment with Linux safely

##### When You‚Äôre Done

* Close the VM window to pause Ubuntu

* You can resume anytime, or take snapshots to save your progress

This is one of the safest and most flexible ways to learn Ubuntu. You get all the power of Linux, with none of the risk.

* **Install Ubuntu from Microsoft Store**

    * Open Microsoft Store

    * Search for "Ubuntu"

    * Click "Get" to install your preferred version (e.g., Ubuntu 22.04 LTS)

* **Launch Ubuntu**

    * After installation, open Ubuntu from the Start Menu

    * Create a username and password when prompted

* **Start Using Ubuntu**

    * You now have a full Linux terminal inside Windows

    * Try commands like:

        * `sudo apt install nano g++` ‚Üí install tools for C++ development

##### What You Can Learn

### 1.2.4 Getting familiar with the terminal and file system

For many Windows and macOS users, the terminal can feel intimidating at first. It‚Äôs just a black screen with blinking text ‚Äî no icons, no buttons, no safety nets. But here‚Äôs the secret: the terminal is where real geeks thrive. It‚Äôs fast, powerful, and gives you direct control over your computer. Once you learn how to use it, you‚Äôll never want to go back.

The terminal lets you talk to your computer using text commands. You can navigate folders, create files, install software, and even compile your C++ programs ‚Äî all from the keyboard. It‚Äôs like having superpowers that most users never unlock.

The file system is how your computer organizes everything: folders, files, programs, and settings. In Linux (Ubuntu), the file system starts at the root / and branches out into directories like /home, /etc, and /usr. You‚Äôll learn how to move through this structure using terminal commands.

Here‚Äôs a list of essential terminal commands every geek kid should know:

#### Navigation

* `pwd` ‚Äî Show your current directory

* `ls` ‚Äî List files and folders

* `cd foldername` ‚Äî Change directory

* `cd ..` ‚Äî Go up one level

* `cd /` ‚Äî Go to the root directory

* `cd ~` ‚Äî Go to your home directory

#### File and Folder Management

* `mkdir foldername` ‚Äî Create a new folder

* `touch filename.txt` ‚Äî Create an empty file

* `cp source.txt destination.txt` ‚Äî Copy a file

* `mv oldname.txt newname.txt` ‚Äî Rename or move a file

* `rm filename.txt` ‚Äî Delete a file

* `rm -r foldername` ‚Äî Delete a folder and its contents

* `tree` ‚Äî Show folder structure (install with `sudo apt install tree`)

#### Viewing and Editing Files

* `cat filename.txt` ‚Äî View file contents

* `less filename.txt` ‚Äî Scroll through file contents

* `nano filename.txt` ‚Äî Edit a file in the terminal

* `gedit filename.txt` ‚Äî Edit a file with a graphical editor

#### System Info and Permissions

* `whoami` ‚Äî Show your username

* `id` ‚Äî Show user and group info

* `chmod +x filename.sh` ‚Äî Make a file executable

* `sudo command` ‚Äî Run a command as superuser

* `top` ‚Äî View running processes

* `htop` ‚Äî Advanced process viewer (install with sudo apt install htop)

* `df -h` ‚Äî Show disk usage

* `free -h` ‚Äî Show memory usage

* `uname -a` ‚Äî Show system info

* `uptime` ‚Äî Show how long the system has been running

* `hostname` ‚Äî Show system hostname

#### Package Management (Ubuntu)

* `sudo apt update` ‚Äî Refresh package list

* `sudo apt upgrade` ‚Äî Install updates

* `sudo apt install packagename` ‚Äî Install a package

* `sudo apt remove packagename` ‚Äî Remove a package

* `sudo apt purge packagename` ‚Äî completely removes a package from your system along with its configuration files

* `apt list --installed` ‚Äî List installed packages

* `dpkg -l` ‚Äî Another way to list packages

#### Search and Help

* `man command` ‚Äî View manual for a command

* `command --help` ‚Äî Show help options

* `grep keyword filename.txt` ‚Äî Search for text inside files

* `find . -name "filename"` ‚Äî Search for files by name

* `which command` ‚Äî Show the path of a command

* `locate filename` ‚Äî Quickly find files (install with `sudo apt install mlocate`)

* `history` ‚Äî Show command history

* `history -c` ‚Äî clear command history

#### Useful Shortcuts

* **Tab** ‚Äî Auto-complete file and folder names

* **Ctrl+C** ‚Äî Cancel a running command

* **Ctrl+L** ‚Äî Clear the terminal screen

* **Up Arrow** ‚Äî Repeat previous command

* **Ctrl+Shift+T** ‚Äî Open a new terminal tab

* **Ctrl+D** ‚Äî Log out or close terminal

Once you master these commands, you‚Äôll be able to navigate, manage, and troubleshoot your system like a real geek. These tools are used everywhere ‚Äî in Ubuntu, in embedded boards, inside Windows 11 (WSL), and in cloud terminals. The terminal is your universal passport to Linux-powered environments.

In the next sections, you‚Äôll use these commands to install build tools, compile C++ code, and manage your projects. So take your time here ‚Äî this is where your geek powers begin.
